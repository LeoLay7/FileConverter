# 2. Исследование и проектирование пользовательского интерфейса

## 2.1. Логика пользовательского сценария

Пользовательский сценарий frontend-части строится вокруг последовательной конвертации файла. Сначала пользователь попадает на главную страницу, где получает краткое описание назначения системы и может перейти к началу работы. Далее выполняется загрузка исходного файла через область выбора или перетаскивания. После выбора файла интерфейс отображает его имя, размер, определенный формат и текущий статус обработки.

Следующим шагом пользователь выбирает целевой формат. После успешной загрузки файл передается в backend, где сохраняется и получает идентификатор. На основании этого идентификатора frontend запрашивает структуру файла и отображает ее в виде дерева полей. Пользователь может раскрывать вложенные элементы, выбирать конкретные поля и просматривать сведения о них.

После анализа структуры пользователь переходит к настройке правил преобразования. В интерфейсе предусмотрена логика работы с операциями над полями: переименование, удаление, изменение типа, копирование значения и применение правила к элементам массива. После настройки правил frontend отправляет конфигурацию на backend, запускает задание конвертации и отображает его статус до завершения. При успешной обработке пользователь получает возможность скачать результирующий файл.

## 2.2. Структура интерфейса

Интерфейс приложения состоит из нескольких основных экранов. Главная страница выполняет вводную функцию: она объясняет назначение сервиса, показывает поддерживаемые форматы и описывает общий порядок работы. Эта страница используется как точка входа в пользовательский сценарий.

Страница загрузки файла отвечает за выбор исходных данных и целевого формата. На ней реализована зона drag-and-drop, выбор файла через системный диалог, отображение информации о выбранном файле, выбор целевого формата и базовая проверка допустимого расширения. Этот экран связывает пользовательское действие с backend API загрузки файла.

Страница структуры файла предназначена для отображения дерева полей. Она содержит панель с информацией о загруженном файле, инструменты раскрытия и сворачивания дерева, список типов данных, область дерева и панель деталей выбранного поля. Этот экран необходим для того, чтобы пользователь понимал структуру исходных данных перед настройкой правил.

## 2.3. Основные UI-решения

Одним из основных решений стало представление процесса конвертации как пошагового сценария. Это снижает сложность интерфейса и позволяет пользователю последовательно выполнять операции, не сталкиваясь сразу со всеми настройками системы. Визуальные блоки отделяют загрузку файла, выбор формата, анализ структуры и дальнейшую настройку правил.

Для загрузки файла используется drag-and-drop компонент. Он поддерживает выбор файла вручную и перетаскивание в область загрузки. Такой подход соответствует привычным пользовательским паттернам работы с файлами и делает первый шаг сценария более удобным.

Структура файла отображается в виде дерева. Для каждого поля показывается имя, тип, путь и пример значения. Вложенные объекты и массивы можно раскрывать и сворачивать. Такой способ отображения особенно важен для работы с JSON и XML, где данные часто имеют несколько уровней вложенности.

Панель деталей выбранного поля позволяет отделить обзор всей структуры от работы с конкретным элементом. Пользователь может выбрать поле в дереве и увидеть его путь, тип данных, пример значения и сведения о вложенных элементах. Это создает основу для последующей настройки правил преобразования.

## 2.4. Результаты этапа проектирования

По итогам проектирования был сформирован frontend-интерфейс, отражающий основной пользовательский сценарий MVP. Были определены ключевые экраны, структура навигации, способы отображения файла и дерева данных, а также визуальная логика подготовки правил конвертации.

Реализованный интерфейс связан с backend API и позволяет выполнять сценарии, предусмотренные техническим заданием. Frontend выступает прикладным слоем, который скрывает от пользователя технические детали микросервисной обработки и предоставляет единый визуальный процесс работы с файлами.
