# 4. Результат

В результате выполнения работ была реализована frontend-часть приложения для файловой конвертации. Интерфейс предоставляет пользователю визуальный сценарий работы с системой и подключен к backend API, обеспечивающему хранение файлов, получение структуры данных, управление правилами и выполнение заданий конвертации.

Приложение реализовано как одностраничный frontend с маршрутизацией между основными экранами. Были созданы главная страница, экран загрузки файла и экран просмотра структуры файла. Навигация позволяет пользователю последовательно перейти от общего описания сервиса к непосредственной работе с файлом.

Главная страница содержит описание назначения системы, поддерживаемые форматы и краткое объяснение порядка работы. Она выполняет роль стартового экрана и помогает пользователю понять, какие действия необходимо выполнить для конвертации данных.

Экран загрузки файла позволяет выбрать файл через системный диалог или перетащить его в специальную область. После выбора отображаются сведения о файле: имя, размер, формат и статус. Также пользователь выбирает целевой формат, в который должен быть преобразован файл. После загрузки frontend получает идентификатор файла от backend и использует его в следующих шагах.

Экран структуры файла отображает древовидное представление данных. Пользователь может раскрывать и сворачивать элементы, выбирать конкретные поля и просматривать информацию о них. Для выбранного поля отображаются путь, тип данных, пример значения и сведения о вложенных элементах. Это позволяет подготовиться к настройке правил преобразования.

В интерфейсе предусмотрены операции, необходимые для дальнейшей работы с правилами: переименование поля, изменение типа, исключение поля и работа с вложенными структурами. Эти операции соответствуют возможностям backend-части и используются при формировании набора правил.

На момент сдачи frontend подключен к API и обеспечивает выполнение основного сценария по техническому заданию. Пользователь может загрузить файл, получить его структуру, настроить преобразование, запустить конвертацию, отслеживать статус задания и скачать результирующий файл.

Итоговый результат представляет собой пользовательский слой MVP, который делает backend-систему доступной для демонстрации и практического использования. Интерфейс не заменяет backend-логику, а предоставляет удобный способ управления ей.
