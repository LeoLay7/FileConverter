# 1. Введение

Frontend-часть проекта предназначена для предоставления пользователю графического интерфейса работы с системой файловой конвертации. Интерфейс обеспечивает выполнение основного пользовательского сценария: переход к конвертации, загрузку исходного файла, выбор целевого формата, просмотр структуры данных, настройку правил преобразования и дальнейшее получение результата обработки.

Разработка frontend была выполнена как часть общего MVP системы. Пользовательский интерфейс проектировался с учетом требований технического задания и структуры backend API. На момент сдачи проекта frontend подключен к backend-сервисам и использует их для загрузки файлов, получения структуры файла, настройки правил, запуска задания на конвертацию и получения результата.

Основной задачей frontend-части являлось создание понятного и последовательного пользовательского пути. Пользователь не должен взаимодействовать напрямую с отдельными backend-компонентами или техническими деталями обработки. Все операции представлены через визуальные шаги: выбор файла, выбор формата, анализ структуры, настройка преобразований и получение результата.

Интерфейс реализован как одностраничное приложение с маршрутизацией между основными экранами. Визуальная структура построена вокруг карточных блоков, панели навигации, drag-and-drop зоны загрузки, дерева структуры файла и панели сведений о выбранном поле. Такой подход позволяет разделить сложный процесс конвертации на несколько понятных этапов.

В рамках проекта frontend играет роль пользовательского слоя над backend-системой. Backend обеспечивает хранение файлов, обработку правил и выполнение конвертации, а frontend предоставляет удобный способ запуска этих операций и контроля их результата. Это позволяет использовать систему не только как набор API, но и как полноценное приложение для демонстрации сценариев, описанных в техническом задании.
